What this class is about?

This course is NOT about learning new C++ syntax. Instead, we will apply what we have learned in CS161 and CS162 to create new abstract data type.

Our goal this term:

  • to learn different data structures
  • to learn algorithms to solve problems
  • to learn how to measure the efficiency of the approaches taken

Data structure vs abstract data type:

  • A data structure specifies how we store the data (e.g. an array, linked list).
  • An abstract data type (ADT) specifies how a new data type behaves: it includes the data and operations that the new data type requires.
  • ADT uses a data structure to store data.

Using classes to build ADTs:

  • We will use the C++ class construct to build abstract data types
  • The data (represented by a data structure) are placed in the private section
  • The operations (what the "client" or application can invoke) are placed in the public section

Seperation of ADT and clients who use ADT:

  • The clients of ADT should not be aware of what data structure is being used (the private section of the class).
    • This means the client program should not be aware that there is a node or a next pointer for a linked list, or an index to an array.
  • This allows an ADT to "plug and play" different data structures, to maximize efficiency without disrupting the client program.

Some important implementation notes for this term:

    • ADT implemented by class should not prompt, or read from the keyboard because the ADT doesn't know if there is anyone sitting at the keyboard. Instead have the client program do the prompt and read from the keyboard, then pass the data in the member functions of ADT.
      • An example:
        • a good design: add(const Data& data)//read the data in the client program, then pass it in
        • a bad one: add() //prompt and read in data in add function
    • You should only use statically allocated arrays for local variables. All arrays in the data section of ADT should be dynamically allocated.
      • An example:
        • use char * name; in data class so that the array contains exactly number of characters needed for name
        • use char name[MAX_LEN]; where MAX_LEN is a constant integer as temporary storage because you don't know how big the name is yet.
    • Keep your data members restricted to just what is necessary
      • For each data member, ask yourself the question "Could this be a local variable to a member function instead?"
      • If the value of the variable does not need to persist from operation to operation, it should NOT be a data member.
      • An example: Linked list implementation
        • head and tail are appropriate data members of the ADT because they are attributes of the list and their values need to be "remembered."
        • current and previous are NOT appropriate data members although they are used in almost every member function. They should be local variables of the member function because they don't describe the "state" of the list and their values don't need to be "remembered."
    • Last but not the least important note:
      • Think about efficiency this term!
      • Only traverse lists when absolutely necessary.
      • Use pass by reference to reduce the information from continually being copied when passing instances of a struct or a class. If you don't want the function to modify the object, use const Data& data
