Some questions to think about:

  • Assume we have data get(int index); function in the list class which will return the data at the position "index". Which implementation (array or linked list) can provide an efficient solution for this newly added function?
  • Assume we want a unsorted list ADT. Where do you want to insert data in an array implementation (the beginning, the end or somewhere in the middle)? How about linked list implementation?
  • Think about the pros and cons to store pointer to the data in the individual cell of the array (or in the individual node of linked list )?
    • for array implementation: data ** dArray;
    • for linked list implementation: struct node {data * item, node * next};

 

What should I do for this module?

  • Review array implementation for list ADT
    • The video "arrayList" shows you how to use a dynamic array to implement a sorted list
      • three constructors are provided: default constructor, the constructor that initialize the list from external data file, the copy constructor.
      • destructor to deallocate the list. Notice that you don't have to deallocate individual data in the list manually because the destructor of each data object will be invoked automatically. The beauty of OOP -- Every object takes care of itself.
      • overload '=' operator. When it copies inidividual element over, it simply uses '=' for each data object. That is why you want to overload '=' in the data class. It makes the clients' work much easier.
      • accessor functions: retrieve, display and getSize are declared as "const". display uses << to display inidividual data object because we have taught data class how to do '<<' by overloading '<<' in the data class.
      • mutator functions: insert, remove. Insert function will check the capacity of the list first. It will "expand" the array if needed. Whenever a new data needs to be added, the list will insert it at the "correct" position to maintain a sorted list. It uses '<' to compare two data objects because we have overloaded '<' in data class. Now you are reaping the benefits of operator overloading and object-oriented programming.
